# openclaw-bot

> ⚠️ **Generated by an AI coding system — experimental.** Every file in this
> repository — code, configuration, and documentation — was written by the same
> multi-agent AI coding system it helps rebuild. It is **experimental**, built for one
> researcher's personal setup, and **may not work as expected** on other machines,
> other OpenClaw versions, or tasks outside the documented assumptions.

Manifest-driven rebuild for the **OpenClaw** research-assistant bot: sanitized config
templates, workspace skills, and lifecycle scripts. It is the OpenClaw component of the
larger [**coding-system-rebuild**](https://github.com/hoanganhduc/coding-system-rebuild)
backup/restore system, which pins and invokes it.

## What's here

- **`REBUILD-MANIFEST.json`** — allowlist manifest classifying every OpenClaw config,
  workspace skill, agent bootstrap dir, systemd unit, and cron job as
  `public-copy` / `public-template` / `private-archive` / `exclude-*`.
- **Lifecycle scripts** — `sync.sh` (live system → sanitized templates), `install.sh`,
  `backup.sh`, `restore.sh`, `deploy.sh`, `test-roundtrip.sh`.
- **Sanitized artifacts** under `config/`, `workspace/`, `agents/`, `systemd/`, … —
  generated by `sync.sh --apply` with secrets redacted, personal IDs masked, and paths
  templated to `{{ USER_HOME }}` / `{{ OPENCLAW_HOME }}`.

## Use

This component is normally driven by the umbrella system (`make install` there clones and
runs it). Standalone:

```bash
./install.sh --prefix "$HOME/.openclaw" --skip-docker --skip-services --skip-config
```

See the umbrella repo's docs for the full picture, and the
[introductory blog post](https://hoanganhduc.github.io/misc/coding-system-rebuild/) for
what the whole system is and how to try it.
