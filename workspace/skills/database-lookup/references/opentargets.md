# Open Targets

- GraphQL endpoint: `https://api.platform.opentargets.org/api/v4/graphql`
- REST base: `https://api.platform.opentargets.org/api/v4`
- Best for: target-disease associations, tractability, safety liabilities, and pathway context
- No API key required
- Prefer POST GraphQL for richer target or disease queries
- Use when the task is target prioritization or disease-association oriented
